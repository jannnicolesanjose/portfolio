KAWAY ASL Helper.

Repository Launch Site

An American Sign Language (ASL) fingerspelling recognition software and interactive learning aid system utilizing Random Forest and computer vision architectures.

// Project Overview

KAWAY is a computer-vision-powered assistive technology platform designed to detect, translate, and transcribe American Sign Language (ASL) fingerspelling gestures in real time[cite: 3]. Built to minimize the communication barrier between sign language users and non-signers, the system features multiple dedicated software pathways including Learn Mode, Practice Mode, and a bidirectional translation suite (ASL-to-Text and Text-to-ASL)[cite: 3, 3].

The desktop web interface enables users to practice basic hand shapes right from their homes[cite: 3]. Through active camera interaction feeds, the algorithm captures raw geometric parameters, processes knuckles via localized landmark matrices, and maps inputs instantly against trained classification structures to print alphabetic counterparts sequentially[cite: 3, 3].

// The Problem & Motivation

Fingerspelling is a critical part of sign communication used frequently to convey names, places, and specific terminologies that lack direct sign language equivalents[cite: 3, 3]. However, due to the high execution speeds and intricate finger variations involved, beginners struggle to perceive static configurations using traditional text guides or peripheral vision[cite: 3].

Furthermore, institutional disparities and a lack of scalable digital learning dictionaries widen the gap for hearing-impaired students[cite: 3, 3]. KAWAY directly overcomes these structural roadblocks by automating feature extraction and processing, providing real-time, zero-cost diagnostic feedback to boost the proficiency and confidence of student signers[cite: 3, 3].

// Random Forest Algorithm

Chosen over heavy convolutional neural network methods to drastically reduce computational load and decrease step execution runtimes without degrading overall precision matrices[cite: 3, 3].

Maps 21 structural hand coordinates based on knuckle intersections in real time, converting spatial orientation arrays into localized feature vectors[cite: 3, 3].

// OpenCV Video Capture Core

Handles real-time video streaming directly from device webcams, executing frame capturing and preprocessing parameters synchronously[cite: 3, 3].

// Flask & Vanilla JavaScript

Wires Python background prediction streams straight into an interactive, cross-platform client frontend shell asynchronously using live route embedding[cite: 3].

divide_and_conquer.js Parallel Text-to-ASL Mapping 
// Divide-and-Conquer algorithm layout to recursively process word splits into character assets
function processLetters(chars) {
    const n = chars.length;
    
    // Base Case execution: Single element canvas population
    if (n === 1) {
        if (chars[0] === ' ') return createSpaceCanvas();
        return loadImageAsset(chars[0]); // Asynchronous parallel promise initialization
    }
    
    // Divide and Conquer step split computation paths
    const mid = Math.floor(n / 2);
    const leftHalf = chars.slice(0, mid);
    const rightHalf = chars.slice(mid);
    
    return Promise.all([processLetters(leftHalf), processLetters(rightHalf)])
        .then(([leftCanvas, rightCanvas]) => mergeCanvases(leftCanvas, rightCanvas));
}

// Challenges & Technical Bottlenecks

A major engineering bottleneck was handling dynamic moving letters like "J" and "Z" within a static landmark modeling layout, alongside stabilizing outputs under varying light constraints and clutter backgrounds[cite: 3, 3]. To isolate noise factors, data coordinates underwent strict spatial normalization parameters before tracking computation[cite: 3].

The final prototype successfully achieved a 100.00% evaluation classification accuracy across a target dataset containing 23,400 customized training frame captures, running at a lightning-fast inference tracking latency benchmark of 0.02 seconds[cite: 3, 3].

// Educational Impact

By providing an accessible, immediate feedback loops without needing external hardware like data gloves, KAWAY offers a scalable digital blueprint for specialized special education tools, expanding assistive computing opportunities within public domains[cite: 3, 3].
