MataMuseo App.

Repository

An interactive mobile museum tour assistant utilizing a lightweight Vision Transformer model for artifact recognition and machine-learning content filtering for relic suggestions.

// Project Overview

MataMuseo is a context-rich mobile application engineered specifically as an automated tour aid for the Museo De La Salle. The application uses a custom-compressed Vision Transformer algorithm running directly on mobile hardware to scan, identify, and classify historical museum artifacts on site without requiring an internet connection.

Once an object is scanned via the camera view, the tool displays an informative digital profile containing details such as name, historical background, materials, and dimension metadata. Simultaneously, a built-in recommendation engine uses feature embeddings to calculate and suggest contextually related relics, offering users an intuitive, self-paced learning experience.

// The Problem & Motivation

As a lifestyle museum, Museo De La Salle meticulously recreates immersive historical environments, such as late 19th-century Ilustrado homes. To maintain this authentic historical setting, individual exhibits lack traditional text labels or descriptive display cards. Consequently, visitors must rely entirely on verbal explanations during group tours.

This approach can lead to information gaps if tour sizes are large or if visitors wish to explore at their own speed. MataMuseo provides a non-intrusive solution to this problem, delivering descriptive textual summaries directly to the visitor's palm without disrupting the museum's design or atmosphere.

// Technical Stack Deep Dive

// MobileViT-XXS Model

A lightweight Vision Transformer architecture trained via Knowledge Distillation from a heavy BEiT teacher model, optimized to fit within a compact 4 MB footprint.

// Content-Based Filtering

Computes item similarities by blending metadata binaries (material, room location) with Sentence Transformer description matrix vectors.

// Flutter Framework (Dart)

Powers the native multi-platform client shell, linking camera permissions and UI navigation views directly to hardware processes.

// TensorFlow Lite Engine

Executes integer-quantized neural model passes directly on the device, allowing real-time object tracking entirely offline.

def calculate_composite_similarity(meta_vector_a, meta_vector_b, text_embed_a, text_embed_b):
    # Core mathematical norm calculation metrics
    cos_meta = cosine_similarity(meta_vector_a, meta_vector_b)
    cos_text = cosine_similarity(text_embed_a, text_embed_b)
    
    # Weight settings prioritizing physical metadata properties over textual semantics
    w_meta = 0.7
    w_text = 0.3
    
    final_similarity_score = (w_meta * cos_meta) + (w_text * cos_text)
    return final_similarity_score

// Challenges & Technical Bottlenecks

Deploying dense transformer layers onto mobile chips requires keeping model files highly compressed while minimizing inter-class confusion among visually similar artifacts (such as matching antique beds or porcelain vases).

Through structured quantization routines, the final application footprint compressed down to a sleek 4 MB file size, delivering an outstanding classification accuracy rating of 98.17% over 124 asset classes with an average hardware execution speed of 122.34 ms.

// Educational Impact

Evaluated across ISO/IEC 25010 standards, user surveys highlighted high system metrics (Visitor Score: 4.70, Admin Score: 4.81), proving that lightweight edge AI provides a reliable method for preserving cultural heritage and supporting independent public education.
