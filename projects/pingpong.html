A physics-driven desktop retro compilation game built using Python to handle coordinate acceleration matrices.

// Project Overview

This standalone retro application is an interactive implementation of the classic arcade ping pong game. Built entirely inside a script-driven environment, the program uses dynamic velocity vectors to simulate smooth arcade ball rebounds and collision checks in real time.

// The Problem & Motivation

Building dynamic games requires a deep understanding of collision mechanics, precise frame rate control, and user input handling loops. This desktop project was created to implement fundamental computer graphics equations, translating basic matrix arithmetic directly into real-time gameplay feedback.

# Calculate real-time ball coordinate updates and box collisions
def update_ball_position(ball_rect, velocity_x, velocity_y):
    ball_rect.x += velocity_x
    ball_rect.y += velocity_y
    
    # Run dynamic bounds calculation for wall rebounds
    if ball_rect.top <= 0 or ball_rect.bottom >= 600:
        velocity_y *= -1  # Reverse vertical direction vector
        
    return velocity_y

// Challenges & Technical Bottlenecks

Keeping game speeds completely uniform across devices with different processor speeds was a major challenge. We fixed this by locking the main update cycle into a strict 60 FPS clock matrix loop, keeping the physics completely stable on any hardware setup.

// Educational Impact

This application demonstrates an elegant implementation of linear physics logic loops, showcasing how simple code blocks can power complex, interactive user experiences cleanly.
